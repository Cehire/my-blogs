# Integer（Byte、Short、Long、Character）的缓存机制

这几个类都是具有缓存机制的类。缓存工作都是在静态块中完成，在类生命周期的初始化阶段执行。

缓存的范围为：

  • Byte、Short、Integer、Long为-128~127

  • Character为0~127

  • 只有Integer可以通过JVM参数指定缓存范围

    Integer的缓存上界high可以通过JVM参数指定，会取指定值与127的最大值，并且不超过Integer的表示范围，但是下界low不能指定，只能为-128

注意：

  • 整型的包装类valueOf方法返回对象时，在缓存范围内，会返回缓存对象
  • 浮点型的包装类valueOf方法返回新的对象
  • 布尔型的包装类valueOf方法返回Boolean类的静态常量TRUE|FALSE

对应包装类调用valueOf()方法即可装箱，对应包装类对象调用基本数据类型Value()方法即可拆箱

# 构造器是否可以被重写

构造器是不能被继承的，因为每个类的类名都不相同，而构造器名称与类名相同，所以根本谈不上继承，由于构造器不能继承，所以不能被重写，但是构造器可以被重载

# == 和 equals的区别

==是等于比较运算符，比较基本类型变量时，比较的是值；比较引用变量时，比较的是两个引用变量是否指向同一对象

equals()是Object类的方法，我们可以重写equals()方法来按照我们指定的规则进行比较

# 对象拷贝（浅克隆和深克隆的区别）

可以实现Cloneable接口并且重写Object类中的clone()方法，浅克隆下，只会克隆对象的引用，实际上这个引用和原引用还是指向同一个对象，e1修改后e2中的属性也会被修改。深克隆下连对象都会复制一份，e1,e2不会互相干扰，要通过Cloneable接口实现深克隆，那么该类中的引用对象属性也需要实现Cloneable接口

也可以实现Serializable接口实现深克隆
